# AgentKit SEO X Twitter

## Overview

Use the X/Twitter hub to improve profile clarity and posting structure while
avoiding overclaims about undocumented live ranking systems.

## Read path

1. Read [references/profile-and-posts.md](references/profile-and-posts.md)
   for bio, profile, pinned-post, media, and thread structure work.
2. Read [references/section-recipes.md](references/section-recipes.md)
   for display-name, bio, pinned-post, single-post, thread, and Alt Text
   drafting patterns.
3. Read [references/engagement-and-ranking.md](references/engagement-and-ranking.md)
   for growth strategy, reply behavior, and ranking-signal explanations.
4. Read [references/premium-and-confidence.md](references/premium-and-confidence.md)
   for Premium-specific advice and confidence labeling.
5. Read [references/account-audit-and-maintenance.md](references/account-audit-and-maintenance.md)
   when auditing an existing account, reviewing performance patterns, or
   planning ongoing maintenance.

## Rules

- Treat Phoenix, Grok, and related architecture clues as design signals, not as
  a complete live-production contract.
- Do not promise ranking outcomes.
- Keep recommendations aligned with the user's actual niche, expertise, and
  posting capacity.
- Distinguish official product features from empirical tactics.
- Keep profile positioning, pinned assets, posting topics, and linked external
  proof aligned around one clear niche.
- When profile proof, audience, or posting history is missing, ask for it
  before inventing claims or forcing a content strategy.

## Output expectation

Produce X/Twitter-ready bio, pinned-post, or posting recommendations that are
practical, specific, and scoped to the user's real goals.
