# AgentKit SEO Web Portfolio

## Overview

Use this skill to improve how a personal site is crawled, rendered,
summarized, and trusted by search engines and AI systems.

## Read path

1. Read [references/indexing-and-architecture.md](references/indexing-and-architecture.md)
   for crawlability, sitemaps, robots, launch, and URL structure.
2. Read [references/metadata-structured-data-and-js.md](references/metadata-structured-data-and-js.md)
   for titles, canonicals, schema, and JS rendering constraints.
3. Read [references/section-recipes.md](references/section-recipes.md)
   for homepage, About page, project-page, metadata, and `llms.txt` writing
   patterns.
4. Read [references/content-performance-and-aeo.md](references/content-performance-and-aeo.md)
   for case-study writing, mobile performance, and AI-retrieval conventions.
5. Read [references/portfolio-audit-and-maintenance.md](references/portfolio-audit-and-maintenance.md)
   when auditing an existing site, reviewing a deployment, or planning ongoing
   maintenance.

## Rules

- Separate documented standards from emerging conventions such as `llms.txt`.
- Prefer changes that improve crawlability, information scent, and snippet
  quality without adding hype.
- Do not present unofficial AI or SEO proposals as universal standards.
- Keep metadata, structured data, and visible copy aligned.
- Keep page purpose, URL structure, internal links, and proof assets aligned so
  every important claim resolves to a crawlable page.
- When facts are missing, ask for the canonical URL, page inventory, or source
  content before inventing portfolio copy or structured data.

## Output expectation

Return web-ready changes or recommendations tied to real pages, metadata, and
site structure rather than vague SEO advice.
