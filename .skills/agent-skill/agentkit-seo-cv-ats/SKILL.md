# AgentKit SEO CV ATS

## Overview

Use only the CV and ATS guidance relevant to the requested deliverable. Keep
the advice conservative, parser-safe, and grounded in documented, durable
constraints.

## Read path

1. Read [references/structure-and-formatting.md](references/structure-and-formatting.md)
   for layout, section order, file-type choice, and ATS-safe structure.
2. Read [references/section-recipes.md](references/section-recipes.md) when
   drafting or rewriting summary, experience, skills, and education sections.
3. Read [references/keywords-and-bullets.md](references/keywords-and-bullets.md)
   for job-description tailoring and bullet rewrites.
4. Read [references/parser-risks-and-agent-workflow.md](references/parser-risks-and-agent-workflow.md)
   for parser-failure diagnosis and plain-text-source workflows.
5. Read [references/cv-audit-and-maintenance.md](references/cv-audit-and-maintenance.md)
   for full-document reviews, consistency checks, and update workflow.

## Rules

- Do not claim guaranteed ATS success or exact ranking behavior.
- Prefer simple structure, plain section names, and measurable outcomes.
- Tailor wording to the target role, but do not fabricate tools, metrics, or
  employers.
- If the user supplies a job description, align terminology to that role while
  preserving the user's real experience.
- Optimize for reliable parsing first, recruiter readability second, and visual
  polish third.
- Preserve factual alignment with the user's context file, LinkedIn, and public
  portfolio.
- For rewrites, improve section clarity and evidence density before changing the
  user's positioning strategy.

## Output expectation

Return edits that are specific to the user's target role and easy to apply to a
real resume or CV draft.
